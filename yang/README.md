# Trend Collector

한국 커뮤니티/유튜브/Google Trends에서 매시간 원문 신호를 수집해, 최종적으로 웹에 노출할 **오늘의 트렌드 단어 10개**를 고르기 위한 내부 수집 파이프라인입니다.

이 프로젝트는 커뮤니티 원문을 보여주는 서비스가 아닙니다. 원문 제목/댓글은 내부 신호 감지용으로만 저장하고, 공개 웹에는 LLM/통계/편집 관문을 통과한 단어 10개만 노출합니다.

## 핵심 아이디어

최종 산출물은 "많이 나온 단어 순위"가 아니라 **오늘 카드로 설명할 가치가 있는 트렌드 단어 10개**입니다.

예를 들어 현재 버킷 로그에서 관찰된 `뚱쭝폴드`는 카드 후보로 적합합니다.

```text
📌 뚱쭝폴드 [화제·신규]
   뜻: 두꺼운 갤럭시 폴드 모델을 놀리듯 부르는 별칭
   근거: 더쿠 핫 게시글에서 제품명/별칭 맥락으로 등장
```

반대로 `찐찐막`, `지린다`, `실환가` 같은 표현은 많이 보여도 대체로 카드 후보로 부적합합니다. 독립적인 화제라기보다 댓글 말투나 반응 표현에 가깝고, 한 장의 카드로 설명할 새 정보가 적기 때문입니다.

## 전체 흐름

```text
1. 매시간 스케줄링 수집
   디시·더쿠·인스티즈·네이트판·유튜브·Google Trends 원문을 raw_items에 버킷 단위로 저장

2. LLM으로 관문 1: 자격 필터
   원문에서 후보 표현을 뽑고, 추임새·흔한말·노이즈를 대부분 제거

3. 관문 2: 점수화
   z-score, 교차출현, 신조어성 판단으로 후보 30개 생성

4. 관문 3: 편집
   후보 30개 중 카드가 되는 단어 10개를 무순위로 선별

5. 공개 웹 노출
   원문 전문이 아니라 최종 단어 10개와 별도로 작성한 설명/근거만 노출
```

현재 구현은 1번, 즉 **수집과 버킷 로그 축적**까지입니다. LLM 추출, z-score, 후보 30개, 최종 카드 10개 생성은 다음 단계입니다.

## 카드가 되는 단어

카드가 되는 단어는 사용자가 봤을 때 10초 안에 "아, 이게 요즘 왜 보이는지" 이해할 수 있는 단어입니다.

조건은 다음과 같습니다.

```text
뜻 + 맥락 + 현재성 + 설명 가치
```

| 표현 | 카드 적합성 | 이유 |
|---|---|---|
| `뚱쭝폴드` | 적합 | 갤럭시 폴드 관련 별칭으로 뜻과 맥락을 설명할 수 있음 |
| `대신쉬었음` | 적합 가능 | 특정 AI 밈/캐릭터 맥락이 있으면 카드화 가능 |
| `마운자로` | 적합 가능 | 다이어트 약 이슈로 설명 가능 |
| `호프` | 적합 가능 | 영화/콘텐츠 고유명사로 설명 가능 |
| `찐찐막` | 대체로 부적합 | "진짜 마지막"이라는 말투성 표현에 가까움 |
| `지린다` | 부적합 | 감탄사라 독립 화제로 보기 어려움 |
| `진짜`, `너무` | 부적합 | baseline이 높은 흔한 말 |

단, 표현형 유행어도 항상 제외되는 것은 아닙니다. 어떤 표현 자체가 다시 밈으로 떠서 "왜 지금 보이는지"를 설명할 수 있으면 카드 후보가 될 수 있습니다.

## 수집 대상

현재 6개 소스를 수집합니다.

| source | 역할 | 수집 단위 |
|---|---|---|
| `dcbest` | 디시 실베. 남초/커뮤니티 밈 신호 | 제목 |
| `theqoo` | 더쿠 핫. 여초/팬덤/이슈 신호 | 제목 |
| `instiz` | 인스티즈. 10·20 여성 커뮤니티 신호 | 제목 |
| `natepann` | 네이트판. 사연/이슈/베플 구어체 신호 | 제목, 댓글 |
| `youtube` | 유튜브 인기 영상과 댓글. 대중 반응 신호 | 제목, 댓글 |
| `gtrends` | Google Trends RSS. 뉴스/대중화 신호 | 제목 |

## DB와 버킷

수집 결과는 SQLite `trend.db`의 `raw_items` 테이블에 저장됩니다.

| 컬럼 | 설명 |
|---|---|
| `source` | 수집 소스 |
| `unit` | `title` 또는 `comment` |
| `text` | 원문 제목/댓글 |
| `text_hash` | 중복 판정용 hash |
| `meta` | 소스별 부가정보 JSON |
| `bucket_at` | 1시간 단위 버킷 시각 |
| `collected_at` | 실제 수집 시각 |

같은 버킷 안에서 같은 원문은 중복 저장하지 않습니다.

```text
UNIQUE(source, text_hash, bucket_at)
```

## 버킷 로그

`npm run sources`를 실행하면 DB 저장과 함께 `logs/sources.log`에 사람이 읽을 수 있는 버킷 로그가 남습니다.

버킷 로그의 목적은 "같은 시간대에 각 소스가 어떤 신호를 뱉었는지"를 빠르게 확인하는 것입니다. 원문 전문을 공개 서비스에 쓰기 위한 로그가 아닙니다.

예시:

```text
==============================================================================
버킷 2026-07-22T14:00:00.000Z  (수집 2026. 7. 22. 오후 11:44:59 KST)
==============================================================================

[dcbest] 제목 49
  · [부갤] 요즘유행하는 대신쉬었음 ai 청년...

[theqoo] 제목 20
  · 오늘 공개된 갤럭시 폴드8(뚱쭝폴드)...
  · <호프> 3차 감상회 또 하며 썰푸는 단군...

[instiz] 제목 44
  · 마운자로는 장기 망가진다 해서 겁나는데
  · 호프 지금 예상 추이가 얼마야??

[youtube] 제목 20, 댓글 120
  · [어벤져스: 둠스데이] 티저 예고편
  💬 찐찐막으로 한번만 더 속아본다
```

이 로그에서 `뚱쭝폴드`, `대신쉬었음`, `마운자로`, `호프`는 후보가 될 수 있지만, `찐찐막`은 보통 관문 1에서 `[추임새]`로 탈락합니다.

## 관문 1: 자격 필터

LLM이 원문에서 후보 단어/표현을 뽑은 뒤, 대부분을 여기서 탈락시킵니다.

| 기준 | 처리 |
|---|---|
| 태그 필터 | `[화제]`, `[고유명사]` 중심 통과. `[추임새]`는 기본 제외 |
| 흔함 제외 | `진짜`, `너무`, `그냥`처럼 baseline이 높은 말은 제외 |
| 노이즈 제외 | 1회성 오타, 영어 팬덤 댓글, 의미 없는 반복 표현 제외 |

## 관문 2: 점수화

관문 1을 통과한 후보에 신호 3개를 더해 후보 30개를 만듭니다.

| 신호 | 근거 문장 예시 |
|---|---|
| z-score, 급증도 | "어제 0회에서 오늘 5버킷 연속 등장" |
| 교차출현 | "디시·더쿠 동시 등장" |
| 신조어성/텍스처 | "10·20 신어 텍스처에 근접" |

점수화는 최종 순위를 정하기 위한 것이 아니라, 최종 편집 후보군을 만들기 위한 자격심사입니다.

## 관문 3: 편집

후보 30개에서 최종 10개를 고릅니다. 여기서는 순수 점수순이 아닙니다.

| 기준 | 설명 |
|---|---|
| 카드성 | 뜻이 한 줄로 설명되고, 유래나 맥락이 흥미로운가 |
| 구성비 | 신규 진입 2~3개, 상승 지속 4~5개, 연령대/소스 커버리지 1~2개 |
| 신선도 | 며칠 동안 이미 노출된 단어는 졸업 처리 |
| 표현 가치 | 사용자가 보고 "요즘 왜 보이는지" 이해할 수 있는가 |

## 공개 정책

공개 웹에는 다음만 노출합니다.

- 최종 단어 10개
- 단어의 분류/배지
- 별도로 작성한 한 줄 뜻
- 집계 기반 근거 문장
- 필요 시 짧고 익명화된 예시

공개하지 않는 것:

- 커뮤니티 제목/댓글 전문
- 유저 닉네임, 계정명, URL
- 원문 검색 기능
- 원문 DB/API

즉 `raw_items`는 공개 콘텐츠 DB가 아니라 내부 신호 감지용 로그입니다.

## 실행 방법

```bash
npm install
cp .env.example .env
```

`.env`에 `YOUTUBE_API_KEY`를 넣습니다.

```bash
npm run sources
npm run report
```

최근 3개 버킷을 보고 싶으면:

```bash
npm run report -- 3
```

## 자동 실행

로컬 맥에서는 launchd가 매시간 `scripts/run-sources.sh`를 실행하는 구조입니다.

```text
~/Library/LaunchAgents/com.trendcollector.sources.plist
→ scripts/run-sources.sh
→ npm run sources
→ trend.db / logs/sources.log
```

주의: 맥이 꺼져 있거나 잠들면 해당 시간대 버킷은 비어 있습니다. 24시간 안정 수집이 필요하면 이후 클라우드로 이전해야 합니다.
