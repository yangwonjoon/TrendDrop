## 현재 상태 (한 줄)
6개 소스(디시·더쿠·인스티즈·**네이트판**·유튜브·Google Trends) 원문을 1시간 버킷으로 `raw_items`에 축적하는 수집기 완성 + **launchd 매시간 자동 실행 가동**. **아직 추출(LLM)·z-score·편집은 미구현.** 지금은 "며칠 쌓기" 단계 — 버킷이 자동으로 쌓이는 중.

## 1차 목표 (현재 범위)
**축 1(발굴) 수집 안정화만 한다.** 6소스에서 원문을 꾸준히·안 깨지고 쌓는 것. 그 다음에야 추출·z-score·편집을 얹는다.
- **축 2(연령대 필터)는 현재 목표에 없음.** "10·20 유행어를 노린다"는 발굴 텍스처일 뿐, 단어별 연령 분류/필터 기능은 안 만든다. → 1차가 안정화되면 그때 "어떻게 필터링할지" 설계.
- **축 3(설명 생성)도 나중.**

## 변천사 (발굴 소스가 어떻게 바뀌어 왔나)

### v0 ❌ 유튜브 인기영상 댓글 → n-gram 키워드 추출
초기 설계. **폐기.**
- 댓글은 세상 얘기가 아니라 그 영상 얘기를 함 (뮤비엔 "노래 좋다"만). 상위 키워드가 `노래·영화·기대`뿐.
- 한글 2~4글자 n-gram은 신조어보다 조사·어미 부스러기(`니다·는데·으로`)를 훨씬 많이 만듦.
- 트렌드 판정(z-score) 미구현이라 절대빈도 순위는 항상 흔한 조각이 1등.

### v1 ❌ Google Trends 급상승 검색어(RSS) 단독
**발굴용으로 폐기** (신호용으로는 v2에서 재활용).
- 실측: 최신 스냅샷이 전부 뉴스·방송(`이혼숙려캠프`, `이란전쟁`, `부동산정책`). 신조어 0건.
- 구조적으로 "검색될 만큼 커진 뉴스성 이슈"만 잡힘 → "슬랑이" 같은 은어는 안 나옴.
- (참고) 이 시기 `trending.ts` + `trend_terms` 테이블 + launchd 매시간 자동실행까지 구성했었음. **v2 전환하며 코드·스케줄러 모두 제거.**

### v2 ✅ (현재) 5소스 커뮤니티·댓글 포트폴리오
- 커뮤니티 스크래핑 + 공식 소스 조합으로 신조어를 원천에서 잡는다.
- 소스 구성·역할은 아래 "확정된 설계 결정"·`COLLECTION.md` 참고.

4. **소스 다변화의 근거** — 5소스 동시 스냅샷 비교 결과 결이 뚜렷이 다름:
   - 인스티즈(`익·마운자로·개개개빡`)와 유튜브 댓글(`찐찐막·실환가·지린다`)이 **10·20 유행어에 가장 근접**.
   - `뚱쭝폴드`처럼 여러 소스에 걸치는 후보가 한 버킷에서도 관찰됨 → 교차출현 신호의 실재 확인.

## 확정된 설계 결정 (대화 근거)

- **발굴 타깃 = 10대·20대 유행어**(텍스처). 디시말·이미 대중화된 밈·오래된 은어는 노림 대상 아님.
- **5소스 포트폴리오.** 기준 둘: ① 성별×연령 커버리지가 겹치지 않을 것, ② 혼자 유지보수 가능한 개수. → 디시(남초)·더쿠(여초 팬덤)·인스티즈(10·20 여성)·유튜브(일반인·안정 기둥)·Google Trends(뉴스/대중화 신호·안정 기둥).
- **네이트판 추가 (2026-07-22, 기존 보류 결정 번복)** — 원래 "커버리지 겹침"으로 보류했으나, 사용자 결정으로 6번째 소스로 편입. 근거: 오늘의 톡 랭킹 한 페이지에 **제목 50 + 베플(베스트 댓글) 25**가 같이 오고, 베플이 유튜브 댓글급 구어체 신호를 줌(사연·이슈 결). 유지비 낮음(UTF-8, 단순 정규식 2개).
- **펨코 보류 유지** — 커버리지가 디시와 겹쳐 어댑터 유지비 대비 이득 약함. 나중 recall 구멍 보이면 추가.
- **Google Trends는 뉴스 노이즈 허용하고 후보 소스로 포함** (사용자 최종 결정). 원래 "신호 전용"으로 좁히려 했으나, 뉴스도 데이터로 두고 소스별 로그로 관찰하기로 함.
- **z-score는 소스별로 따로 계산** — 디시 baseline과 인스티즈 baseline은 말투가 완전히 달라, 합치면 v0의 "니다·는데" 함정이 소스 단위로 재발. 소스마다 급증 후보를 뽑고, **합칠 때 교차출현(2+ 소스 동시 등장) 가중치**로 방언 필터를 발굴 단계에 내장.
- **점수는 자격심사, 선정은 편집.** 파이프라인 점수는 "후보 30개"까지만. 최종 10개는 **LLM 편집 패스**가 "이 단어로 좋은 카드를 만들 수 있는가"(뜻 설명 가능·유래 재밌음) 기준으로 고름.
- **순위 매기지 않기.** 1~10위 줄세우기 대신 무순위 카드 + 배지(`🆕 신규`, `📈 3일째 상승`). 데이터가 지지하는 만큼만 주장(= "실시간"→"시간별"과 같은 정직함).
- **신선도 관리** — 같은 단어 노출 상한(예: 5일) 후 강제 졸업. 구성비 가이드: 신규 진입 2~3 / 상승 지속 4~5 / 연령대 커버리지 1~2.
- **카드 스키마(예정)**: 단어 / 한 줄 뜻 / 예문 1개 / 주 사용 연령대 / 근거("디시·더쿠 동시 급증, 3일째 상승").
- **"실시간"이 아니라 "시간별 트렌드"** — 지연 사슬(실베 자체 집계 + 시간별 스크랩 + z-score baseline + 맥 슬립) 때문. 신조어는 며칠 단위로 자라므로 시간별로 충분. 필요 시 15분까지 당길 수 있으나 봇 차단 위험 ↑.
