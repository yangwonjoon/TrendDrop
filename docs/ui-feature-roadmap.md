## 바이럴 / 공유 플랜 (구현은 추후, 설계만)

소셜 트렌드 서비스인데 **공유 루프가 아예 없는 것이 현재 가장 큰 구멍**이다.

### 1. 트렌드 카드 이미지 공유 — 우선순위 높음

- 오늘의 TOP5 또는 특정 키워드를 **이미지 카드로 생성**해 인스타 스토리·카톡에 공유
- 구현: **Canvas API로 클라이언트에서 생성** → `canvas.toBlob()` → `navigator.share()`(Web Share API),
  미지원 브라우저는 다운로드로 폴백. **외부 라이브러리 불필요**
- 규격 2종: 스토리 1080×1920 / 피드 1080×1080
- 톤은 다크 애널리틱스 그대로(모노 숫자, 그린 델타) — 공유될수록 브랜드가 각인됨

### 2. 동적 OG 이미지

- Next.js 내장 `ImageResponse`(`next/og`) 사용 — 별도 설치 없음
- `opengraph-image.tsx`로 순위·상승률·미니 차트가 박힌 링크 미리보기 생성
- 카톡·슬랙·X에 링크를 붙였을 때 카드가 뜨게 → 클릭률 상승
- **주의**: Edge 런타임 + 한글 폰트 서브셋 임베딩이 필요(기본 폰트로는 한글이 깨짐)

### 3. 주간 트렌드 리포트

- 이번 주 급상승 TOP10을 카드뉴스 형태 페이지로 (`/report/[week]`)
- 정적 생성 가능 → **SEO 유입**에도 유리. 이후 Search Console 연결로 성과 측정
  (`google-api-research.md`의 3순위 항목과 연결)

### 측정

- 공유 버튼 클릭 / 이미지 생성 완료 / OG 노출 → 어떤 트렌드가 잘 퍼지는지 추적

## 알려진 한계 (의도된 프로토타입 상태)

- **랭킹 20행이 전부 같은 `/trend`로 연결됨** — 상세 페이지가 1개뿐.
  실제로는 `/trend/[slug]` 동적 라우트 + 키워드별 데이터가 필요
- **데이터가 전부 mock** — 실데이터 연결은 `google-api-research.md` 결론대로
  YouTube Data API를 1순위로, 서버 배치 수집 + 캐시 구조로
- 목록에서 **사라지는 행의 fade-out 미구현** — React가 노드를 제거한 뒤라 애니메이션 대상이 없음.
  퇴장 큐 상태관리가 필요해 비용 대비 이득이 낮다고 판단해 보류
- 타임머신의 가장 오래된 라벨은 "11시간 전" — 12개 스냅샷을 1시간 간격으로 두면 구간이 11시간이라 산술상 정상

## 지켜온 기술 원칙 (계속 유지할 것)

1. **외부 라이브러리 추가 금지** — Web Animations API / `requestAnimationFrame` / 인라인 SVG / CSS만으로 구현
2. **2테마 토큰 시스템** — 토큰 블록 밖 하드코딩 색상 0건. 새 색이 필요하면 토큰을 먼저 추가
3. **하이드레이션 안전** — 렌더 중 `Math.random` / `Date.now` / `new Date` 금지.
   시계열은 고정 시드 PRNG로 생성해 서버·클라이언트 결과 일치(독립 빌드 2회 해시 비교로 검증)
4. **접근성** — 네이티브 컨트롤 우선, `aria-*` 속성, 키보드 완결, 포커스 트랩·복귀
5. **`prefers-reduced-motion`** — 모션만 제거하고 기능은 유지
6. **품질 게이트** — `tsc` / `lint` / `build` 전부 통과 상태를 항상 유지
