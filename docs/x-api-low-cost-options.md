# X 트렌드 분석 — 무료 & 월 $30 이하 저비용 옵션 조사

- **조사일**: 2026-07-08
- **전제**: 본편 조사([x-api-research.md](x-api-research.md))의 "Trends 엔드포인트 중심 + 검색 최소화" 아키텍처 기준
- **결론 먼저**: **월 $30 이하는 가능하다. 단, X 데이터의 "완전 무료" 경로는 없다.** 추천 순서는 **A(공식 API 저용량 설계, 한국 전용 월 ~$19~22) → B(TwitterAPI.io 종량제, 월 $5~27) → D(무료 대체 소스 혼합, $0)**. C(무료 스크래핑)는 2026년 현재 사실상 사망 상태라 비추천.

> ⚠️ **정정 (2026-07-08 재검증)**: 초기 조사에서 언급한 "$500 무료 크레딧"은 **선결제 요구액이 아니라 X가 무료로 주던 바우처**였다. 그러나 클로즈드 베타(2025-10) 참가자 한정이었고, X 직원이 개발자 포럼에서 **"베타 종료, 더 이상 프로모션/바우처 지급 없음"을 공식 확인**(2026-03-06). 서드파티 블로그의 "신규 자동 지급" 정보는 낡은 것. **현재 신규 가입자는 무료 크레딧 없이 실비 충전으로 시작해야 한다** (최소 충전액 강제는 없음).

## 경로 A. 공식 X API 저용량 설계 — 월 $15~29 ★추천 1순위

무료 크레딧은 없지만, 결제 수단 등록 후 소액만 충전하면 된다(구독료·최소 충전액 없음, 청구 주기당 지출 상한 설정 가능 → 상한 도달 시 호출 자동 차단으로 예산 초과 방지).

### 월 $30 이하 설계 예시 — 한국 전용 (상세: [x-trends-korea-only.md](x-trends-korea-only.md))

| 구성 요소 | 사용량 | 월 비용 |
|---|---|---|
| Trends by WOEID 폴링 (한국 23424868, 시간당 1회) | 720 요청/월 | ~$7.2 (요청당 ~$0.01 가정) |
| 상위 트렌드 토픽 대표 트윗 수집 (하루 ~100 read) | 3,000 read/월 | $15.0 ($0.005/read) |
| **합계** | | **~$22/월** |

- 폴링 주기를 2시간으로 늘리면 트렌드 비용은 $3.6 → **합계 ~$19/월**까지 하락. 반대로 15분 주기로 조여도 $28.8이라 $30 안에는 듬
- 비용의 대부분은 트렌드 폴링이 아니라 **대표 트윗 read 수집**($15) — 토픽당 수집 건수를 줄이는 쪽이 예산 조절 레버
- 24시간 중복 제거 덕에 같은 인기 트윗 반복 수집은 과금되지 않아 실비용은 더 낮아질 수 있음
- Trends 요청 단가는 콘솔 단가표에서 실측 확인 필요 — 서드파티 자료 기준 ~$0.01/요청 가정치

**장점**: 완전 합법, 공식 Trends 데이터(tweet_count 포함), 컴플라이언스 엔드포인트 제공, 지출 상한 설정으로 예산 통제 가능
**주의**: 무료 크레딧 프로모션은 종료됨(2026-03 X 직원 확인) — 처음부터 실비 과금. 가입 시점에 콘솔에서 프로모션 부활 여부만 확인해볼 것.

## 경로 B. TwitterAPI.io 종량제 — 월 $30로 트윗 20만 건 ★추천 2순위 (리스크 감수 시)

직접 조회로 확인한 단가(2026-07-08): **트윗 $0.15/1,000건, 프로필 $0.18/1,000건**, 호출당 최소 15크레딧($0.00015), 최소 결제액 없음, 충전 크레딧 무기한. 가입 시 소액($0.1~1) 무료 크레딧 제공. 트렌드 포함 75+ 엔드포인트 제공.

### 월 $30 예산 시뮬레이션

| 구성 | 사용량 | 월 비용 |
|---|---|---|
| 트렌드 폴링 (5개 지역, 시간당) | 3,600 호출 | ~$0.54 |
| 키워드/해시태그 검색 수집 | 150,000 트윗 | $22.50 |
| 유저 프로필 보강 | 20,000명 | $3.60 |
| **합계** | **월 15만 트윗 + α** | **~$27/월** |

같은 예산으로 공식 API 대비 **약 30배의 데이터량**. 월 $5만 써도(트윗 3만 건) 소규모 트렌드 사이트는 충분히 돌아간다.

**장점**: 압도적 가성비, 캡 없음, full-archive 유사 기능
**리스크 (본편 §5와 동일 — 반드시 인지)**:
- 스크래핑 기반 = X 약관 위반 영역에서 동작. 공급 중단이 언제든 가능
- 삭제 트윗 동기화 수단 없음 → 원문 장기 저장 시 개인정보 리스크는 자체 부담
- 수익화 규모가 커질수록 법적 노출 증가. **프로토타입/검증용으로는 합리적, 프로덕션 단일 의존은 비권장**
- 유사 업체(Sorsa, Postproxy, ScrapeCreators 등)도 $0.05~0.30/1,000건대 — 한 곳에 락인되지 않게 어댑터 계층을 두면 공급 중단 리스크 완화 가능

## 경로 C. 무료 스크래핑 도구 — ❌ 2026년 현재 사실상 불가

| 도구 | 2026년 상태 |
|---|---|
| snscrape | **사망** — 2023년 X가 비로그인 게스트 접근 차단 후 작동 불가, 개발 중단 |
| Nitter (공개 인스턴스) | **대부분 폐쇄** — 게스트 계정 차단으로 운영 불가, 남은 인스턴스도 rate limit으로 실사용 불가 |
| Twint | **사망** (개발 중단 수년 경과) |
| twscrape | 유일하게 개발 활발하나 **본인 X 계정 로그인 기반** — 계정 밴 위험 상시, 대규모 수집엔 취약한 계정 풀 필요. 상업 서비스 기반으로는 부적합 |

**결론**: "무료 + 안정적" X 데이터 수집 수단은 2026년 현재 존재하지 않는다. 무료가 필수라면 경로 A의 무료 크레딧이나 경로 D가 현실적 답.

## 경로 D. 무료 대체/보조 데이터 소스 — "X만 고집하지 않는다면"

트렌드 분석 사이트가 목적이라면, X 데이터를 보완·대체할 완전 무료 소스가 있다:

| 소스 | 비용 | 제공 데이터 | 비고 |
|---|---|---|---|
| **Bluesky API / Jetstream firehose** | **완전 무료** (유료 티어 자체가 없음) | 전체 공개 포스트 실시간 스트림, 검색, 프로필 | 인증 없이 firehose 구독 가능. 시간당 5,000포인트 rate limit만 존재. 직접 트렌드 집계 구현 가능. 단 이용자 규모가 X보다 훨씬 작아 한국어 트렌드 커버리지 제한적 |
| **Reddit API** | 비상업 무료 (100 QPS) | 서브레딧 인기글, 급상승 토픽 | **상업 서비스는 유료**($0.24/1,000호출) + 2025년 말부터 신규 OAuth 수동 승인제. 상업 전환 시점에 비용 발생 주의 |
| **Google Trends API (공식 alpha)** | 무료 (승인제) | 검색어 트렌드 | 2025-07 알파 공개됐으나 **테스터 소수만 접근 가능**, 정식 공개는 2027년 이후 전망. 당장은 비공식 라이브러리(pytrends 계열)나 SerpApi 등 유료 우회뿐 |
| **RSS/뉴스 집계** | 무료 | 뉴스 기반 토픽 | X 트렌드의 선행/후행 지표로 보조 활용 |

**현실적 하이브리드 (월 ~$15)**: X 공식 Trends 폴링만 최소한으로 유지(월 $14~15)해 "지금 X에서 뜨는 것"을 가져오고, 상세 볼륨·대화 샘플은 Bluesky firehose(무료)와 Reddit(무료)에서 보강. X read 과금을 0에 가깝게 억제하는 구조. **완전 $0이 필수라면 X 데이터를 포기하고 D만으로 시작하는 수밖에 없다.**

## 추천 로드맵

1. **예산 월 $30가 가능하면**: 공식 API로 한국 전용 Trends 중심 MVP 구축(A, 월 ~$22) — 지출 상한을 $30로 설정해 예산 초과 원천 차단
2. **동시에**: Bluesky firehose 수집기를 무료로 병행 구축(D) — 데이터 다변화 + X 비용 억제
3. **완전 $0이 필수라면**: D만으로 시작(Bluesky + Reddit + 뉴스 RSS 기반 트렌드). X 데이터는 수익/예산이 생긴 뒤 A로 추가
4. **더 많은 X 데이터가 싸게 필요하면**: TwitterAPI.io(B)를 리스크 인지하에 보조로 — 프로토타입 검증엔 소액($5)으로도 충분
5. **하지 말 것**: 무료 스크래핑 도구 기반 설계(C), 서드파티 단일 의존 프로덕션

## 주의사항

- **$500 무료 크레딧은 종료된 프로모션** (베타 한정, 2026-03 X 직원 확인) — 이를 "자동 지급"이라 쓰는 2026년 블로그가 많으니 주의. 선결제 강제액이 아니라 원래도 무료 바우처였음
- TwitterAPI.io 단가, X 공식 단가는 모두 **변동 가능** — 착수 시점에 재확인 필수
- 어떤 경로든 X 원문을 장기 저장하면 삭제 동기화 의무(본편 §4.4)가 따라옴 — 집계 중심 설계 유지
- Reddit은 "비상업 무료"라서 광고/과금이 붙는 순간 유료 계약 대상이 됨
